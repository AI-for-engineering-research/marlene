Branch --c14-substitute M15 | I5
Selects which VIIRS thermal band substitutes for ABI C14 (11.2 µm). M15 (10.76 µm) is the default; I5 (11.45 µm) is a closer spectral match but requires a separate IMG geolocation file. Both paths share identical code — they differ only in which pre-computed normalization stats file is loaded and which output subdirectory (output/features/M15/ vs output/features/I5/) is written to. This walkthrough follows both paths in parallel.
M15 · 10.76 µm

Extract Load MAnet; run frozen encoder+decoder on each train scene ▶

Loads marvelous-moth-137 — an smp.MAnet with a ResNet-18 encoder (pretrained on ImageNet), 5 encoder stages, and decoder channels [256, 128, 64, 32, 16]. Each training scene is processed in a worker pool (worker count = os.sched_getaffinity(0), respecting SLURM limits). The encoder and decoder run under torch.no_grad(); the segmentation head is never called here.

Each scene's three input channels are z-score normalized, padded to the nearest multiple of 32, then forwarded through encoder and decoder only. Padding is stripped before saving.

Check Optional: PCA + logistic regression to verify linear separability ▶

Runs before committing to training. Loads saved feature maps, reshapes each scene to (H×W, 16) pixel vectors, and splits by label > 0 (contrail) vs. background. Up to 50,000 pixels per class are sampled (seeded RNG for reproducibility). If the frozen decoder features are already linearly separable in this 16-dimensional space, training only the segmentation head — which is effectively a linear classifier — is sufficient.

Dataset Build PatchDataset: scan scenes, keep only contrail-positive patches ▶

Loads all training scene .npy pairs from FEATURES_DIR / c14_substitute. PatchDataset pre-scans every valid (top, left) patch position at construction time using a stride of 512 px on 1024 px patches. Only positions where the label mask contains at least 50 contrail pixels are retained — this filters out background-only patches and keeps __len__ exact before any epoch begins.

Freeze Freeze entire network; unfreeze segmentation_head; keep model.eval() ▶

All parameters in the MAnet are frozen first; then segmentation_head parameters are selectively unfrozen. The model is kept in eval() mode for the entire training run — this is intentional: it prevents the frozen encoder's BatchNorm layers from accumulating VIIRS-domain batch statistics, which would corrupt the pretrained ImageNet running means and variances.

L267    param.requires_grad = False
L268for param in model.segmentation_head.parameters():

Adam is constructed on segmentation_head.parameters() only (default lr=1e-3). The segmentation head is a single Conv2d(16 → 2, kernel_size=1) mapping the 16 decoder features per pixel to 2 class logits.

Focal Forward through segmentation_head; compute focal loss ▶

Each mini-batch of pre-extracted features (B, 16, 1024, 1024) is passed directly into model.segmentation_head, which produces logits of shape (B, 2, 1024, 1024) — one score per class per pixel. Focal loss down-weights easy, well-classified pixels (background dominates by count) and focuses gradient on uncertain ones.

$$ \mathcal{L}_\text{focal} = \frac{1}{N}\sum_i \alpha\,(1 - p_{t,i})^\gamma \cdot \text{CE}_i $$

where $p_{t,i} = e^{-\text{CE}_i}$,   $\alpha = 0.25$,   $\gamma = 2$

Eval Load best checkpoint; run full end-to-end inference on val scenes ▶

Loads the best checkpoint into a fresh model instance and sets model.eval(). For each validation scene, the full pipeline runs end-to-end from raw L1B data: band mapping, normalization, padding, full MAnet forward pass (encoder + decoder + segmentation head), softmax, and unpadding. This returns a raw probability map in [0, 1] per pixel; no threshold is applied here so the caller can sweep thresholds independently.

Metrics PR/ROC curves and per-contrail threshold sweep; write metrics JSON ▶

Concatenates probability maps and binary labels from all validation scenes. Computes pixel-level precision-recall and ROC curves via sklearn, then runs a per-contrail threshold sweep: for each labeled contrail polygon (identified by ID k), it measures what fraction of that contrail's pixels exceed each of 20 evenly-spaced thresholds from 0.05 to 0.95. This is more informative than pixel-level metrics alone — a contrail can be partially or fully detected.

Ref Logits → probability: how softmax works

The model outputs two raw logit scores per pixel — one for background ($z_0$), one for contrail ($z_1$). Softmax normalizes these to probabilities that sum to 1. Only the contrail channel (index 1) is extracted as the output probability map.

$$ p_\text{contrail} = \frac{e^{z_1}}{e^{z_0} + e^{z_1}} $$ 
L133probs = nn.functional.softmax(model(pad(torch.Tensor(array))), dim=1)
L136result = probs.cpu().numpy()[0, 1, hedge:-hedge, wedge:-wedge]

dim=1 is the class axis. No threshold is applied by detect_contrails_viirs — the raw probability map is returned so callers can sweep thresholds in the evaluate step.
